## What is LeadLens?

LeadLens is an AI-powered inbound lead intelligence dashboard. Sales teams and business owners paste inbound lead emails → Claude analyzes them → returns a **quality score**, **intent signals**, a **drafted reply**, and **next steps**. Every analysis is saved to Supabase and displayed in a live pipeline dashboard with stats, filters, and CSV export.

### Complementary to Leadify
- **Leadify** handles **outbound** — finding and contacting cold prospects
- **LeadLens** handles **inbound** — scoring and responding to leads who already reached out
- Together they cover the full sales pipeline

---

## Features

- 🎯 **Lead Analyzer** — Paste any inbound email and get an instant AI analysis
- 🔢 **Quality Score** — 1–100 score with animated ring visualization
- 🏷 **Tier Classification** — Hot / Warm / Cold with color-coded badges
- 📊 **Signal Chips** — Intent, Urgency, and Budget Signal at a glance
- ✉️ **Drafted Reply** — Full personalized email reply generated by Claude
- 📋 **Next Steps** — 4 specific action items to close the deal
- 📈 **Pipeline Dashboard** — All leads with stats, filters, and sortable table
- 🔍 **Smart Filters** — Filter by tier, date range, and text search
- 📤 **CSV Export** — Download filtered leads as a spreadsheet

## Getting Started

### Prerequisites
- Node.js 18+
- A [Supabase](https://supabase.com) project
- An [Anthropic](https://console.anthropic.com) API key

### 1. Clone the repo
```bash
git clone https://github.com/your-username/leadlens-ai.git
cd leadlens-ai
```

### 2. Install dependencies
```bash
npm install
```

### 3. Set up environment variables
```bash
cp .env.local.example .env.local
```

Fill in your `.env.local`:
```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ANTHROPIC_API_KEY=your-anthropic-key
```

### 4. Run the migration
Apply the `ll_leads` and `ll_settings` table migration via Supabase dashboard or CLI.

### 5. Start dev server
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Database Schema

### `ll_leads`
| Column | Type | Description |
|---|---|---|
| id | uuid | Primary key |
| business_type | text | Type of business (e.g. Marketing Agency) |
| brand_name | text | The brand being analyzed for |
| email_content | text | The pasted inbound email |
| score | integer | AI quality score (1–100) |
| tier | text | Hot / Warm / Cold |
| reason | text | 2-sentence score explanation |
| intent | text | High / Medium / Low |
| urgency | text | High / Medium / Low |
| budget_signal | text | Strong / Moderate / Weak / None |
| drafted_reply | text | Full AI-drafted email reply |
| next_steps | text[] | Array of 4 action items |
| created_at | timestamptz | Record timestamp |

### `ll_settings`
| Column | Type | Description |
|---|---|---|
| id | uuid | Primary key (single row) |
| default_business_type | text | Pre-fill dropdown default |
| default_brand_name | text | Pre-fill brand name default |
| updated_at | timestamptz | Last updated |
